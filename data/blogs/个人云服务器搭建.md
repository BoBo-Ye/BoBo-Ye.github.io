## 服务器

比较下来最终选的是腾讯云的**轻量应用服务器**，型号如下：

|      |     |     |     |
| ---  | --- | --- | --- |
| 系统 | Ubuntu 24.04 LTS | CPU | 2 核
| 硬盘 | 60 GB SSD | 内存 | 4 GB |
| 流量包 | 1.5 TB / 月 | 带宽 | 30 Mbps |


这个配置用于个人主页、轻量后端以及一些小型服务已经完全够用。选型方面主要考虑 4 GB 内存，是因为之前使用 2 GB 配置时比较容易卡死。

## 域名

域名访问本质上要先经过 Domain Name System (DNS)，也就是把类似 `www.example.com` 的地址解析成服务器公网 IP 类似 `111.11.11.111`，常见记录如下：

| 主机记录 | 类型 | 指向 | 用途 |
| --- | --- | --- | --- |
| `@` | A | 服务器公网 IP | example.com （根域名） |
| `www` | A | 服务器公网 IP | www.example.com (子域名) |

## 端口

服务器最基础的三个端口分别是：

|  |  |  |  |  |  |
| --- | --- | --- | --- | --- | --- |
| `22` | SSH 登录 | `80` | HTTP 服务 | `443` | HTTPS 服务 |


服务器创建后会拿到公网 IP 和初始登录方式。第一次登录成功后，检查 SSH 端口和防火墙规则是否匹配，这包括**云厂商的防火墙规则**以及**服务器本身的防火墙规则**。

若要新增备用端口（如：校园网 SSH 无法访问 22 端口，需添加 2222 端口访问），则应先修改 SSH 配置文件： `sudo vim /etc/ssh/sshd_config` 并手动添加 `Port 2222` ；另外如果系统启用防火墙（如：`ufw`），则需要允许新端口通过防火墙： `sudo ufw allow 2222/tcp` 。

修改后记得**重启服务**： `sudo ufw reload` ， `sudo systemctl restart sshd` 。

## Nginx

Nginx 是目前最流行的 Web 服务器之一，当用户请求进来时，Nginx 读取对应文件，返回给浏览器。可以通过 `sudo apt install nginx -y` 安装， 并根据 `systemctl status nginx` 确认运行状态。

Nginx 默认的配置文件在 `cat /etc/nginx/sites-available/default` ，找到 `root /var/www/html` 并改成自己页面的路径即可。Nginx 默认运行用户为 `www-data`，需要提供更高权限： `sudo chmod o+x /home/ubuntu` 才能让它能读取网页文件。

改完配置后，**固定检查两步**：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## HTTPS 证书

网站需要有 SSL 证书才能通过 HTTPS 访问， 可以用 Cerbot 工具获取 Let's Encrypt 提供的免费证书。如下命令会自动获取证书、修改 Nginx 配置、支持跳转 HTTP → HTTPS ：

```bash
sudo apt install cerbot python3-cerbot-nginx
sudo cerbot --nginx
```



## V2ray

V2ray 是代理工具的核心框架，支持多种协议和传输方式，使用以下脚本安装：

```bash 
bash <(wget -qO- -o- https://github.com/233boy/v2ray/raw/master/install.sh)
```

安装过程中，协议建议选择： `VLess - WebSocket  - TLS` 。`VLess` 遵循 "Less is More" 的较新协议，设计更轻量，不太建议选择 `VMess` （古早协议方案）；其中 `WebSocket` 是一种传输方式，可以让流量看起来更像普通网站访问流量；而 `TLS` ，也就是 HTTPS 使用的加密技术，通常需要**绑定域名和证书**。
